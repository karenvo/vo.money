# XSOAR Series: Getting Started with Cortex XSOAR

**START HERE** - This series covers everything you need to know about Cortex XSOAR (formerly Demisto), from understanding what it is to using it effectively as both an analyst and an engineer.

## Series Overview

This series is designed to be read in order. Each article builds on the previous ones, creating a comprehensive understanding of XSOAR.

## Article Order

1. **[What is XSOAR (and What Makes a Good Automation Engineer)](What-is-XSOAR.md)** (Part 1)
   - Target audience: Security professionals curious about SOAR, managers evaluating tools, analysts wondering if automation engineering is a career path
   - Covers: The problem SOAR solves, what XSOAR actually is, what makes a good automation engineer, when SOAR works (and when it doesn't)

2. **[The Core Components of XSOAR](Core-Components-of-XSOAR.md)** (Part 2)
   - Target audience: Analysts and engineers new to XSOAR who need to learn the vocabulary and mental model
   - Covers: Incidents, indicators, integrations, playbooks, scripts, jobs, lists, and system configuration

3. **[Working as an Analyst in XSOAR](Working-as-an-Analyst-in-XSOAR.md)** (Part 3)
   - Target audience: SOC analysts who need to use XSOAR daily for investigations
   - Covers: The incident queue, war rooms, running commands manually, understanding playbooks, sandboxes, and the investigation workflow

4. **[Context Demystified](Context-Demystified.md)** (Part 4)
   - Target audience: Engineers building playbooks, analysts who want to understand how data flows
   - Covers: What context is, incident context vs playbook context, reading context syntax, subplaybook context, common mistakes, debugging, and best practices

## How to Use This Series

- **New to XSOAR?** Start with Part 1 and read through sequentially
- **Evaluating XSOAR?** Read Part 1 to understand what XSOAR is and when it works
- **Starting as an Analyst?** Read Parts 1-3 for the fundamentals you'll need daily
- **Building Playbooks?** Make sure you understand Parts 1-2, then focus on Part 4 for context
